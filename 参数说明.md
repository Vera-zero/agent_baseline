## 方法二：Zero-Shot Chain-of-Thought

### 数据配置参数 (data)

| 参数路径 | 类型 | 默认值 | 说明 |
|----------|------|--------|------|
| `data.corpus_path` | str | - | 语料库文件路径 (JSON 格式) |
| `data.question_arrow_path` | str | - | 问题数据文件路径 (Arrow 格式) |

### 模型配置参数 (model)

| 参数路径 | 类型 | 默认值 | 说明 |
|----------|------|--------|------|
| `model.provider` | str | "vllm" | 模型提供者，可选值：local (HuggingFace), remote (API), vllm (本地 VLLM 服务) |
| `model.model_name` | str | "qwen3-32b" | 本地模型名称 (用于 local/vllm provider) |
| `model.model` | str | "deepseek-chat" | 远程模型名称 (用于 remote provider) |
| `model.base_url` | str | "http://localhost:8000/v1" | API 服务基础 URL |
| `model.api_key_env` | str | "DEEPSEEK_API_KEY" | API key 对应的环境变量名 |
| `model.device` | str | "auto" | 设备选择，可选值：auto, cpu, cuda |
| `model.torch_dtype` | str | "auto" | PyTorch 数据类型，可选值：auto, float16, bfloat16, float32 |
| `model.max_new_tokens` | int | 256 | 模型生成的最大 token 数量 |
| `model.temperature` | float | 0.0 | 生成温度，0.0 表示贪婪解码 |
| `model.max_retries` | int | 3 | API 调用失败时的最大重试次数 |
| `model.timeout` | int | 180 | API 调用超时时间（秒） |
| `model.batch_size` | int | 1 | 批处理大小（非交互式策略） |

### 检索器配置参数 (retriever)

| 参数路径 | 类型 | 默认值 | 说明 |
|----------|------|--------|------|
| `retriever.type` | str | "contriever" | 检索器类型 |
| `retriever.model_name` | str | "facebook/contriever" | 检索器模型名称 |
| `retriever.top_k` | int | 5 | 检索返回的文档数量 |

### 分块配置参数 (chunk)

| 参数路径 | 类型 | 默认值 | 说明 |
|----------|------|--------|------|
| `chunk.chunk_size` | int | 1200 | 文档分块大小（token） |
| `chunk.chunk_overlap` | int | 100 | 分块之间的重叠大小（token） |
| `chunk.min_chunk_size` | int | 500 | 最小分块大小（token） |

### 运行配置参数 (run)

| 参数路径 | 类型 | 默认值 | 说明 |
|----------|------|--------|------|
| `run.strategy` | str | "zero_shot_cot" | 运行策略，可选值：zero_shot_cot, rag_cot, react |
| `run.max_questions` | int | 100 | 最大处理问题数量，<=0 表示处理全部 |
| `run.resume` | bool | True | 是否从上次中断处恢复运行 |
| `run.save_every` | int | 1 | 每处理 N 个问题保存一次结果 |
| `run.seed` | int | 42 | 随机种子，用于结果可复现 |
| `run.load_strategy_config` | bool | True | 是否加载策略特定配置文件 |
| `run.strategy_config_dir` | str | "./configs/methods" | 策略配置文件目录 |
| `run.strategy_params` | dict | {} | 策略特定参数字典 |

### IO 配置参数 (io)

| 参数路径 | 类型 | 默认值 | 说明 |

## 方法三：RAG Chain-of-Thought

RAG CoT 方法**继承 Zero-Shot CoT 的所有参数**，并实际使用以下额外配置：

### 必需配置差异

| 配置项 | 值 | 说明 |
|--------|-----|------|
| `run.strategy` | "rag_cot" | 必须设置为 rag_cot |
| `retriever.*` | （启用） | 必须配置检索器参数 |
| `chunk.*` | （启用） | 必须配置分块参数 |

### 运行特点

- **需要检索器**：使用 `retriever.type` 和 `retriever.model_name` 配置的检索模型
- **需要语料库**：必须提供有效的 `data.corpus_path`
- **使用分块**：根据 `chunk.*` 参数对文档进行分块处理
- **上下文数量**：由 `retriever.top_k` 控制检索的文档数量

### 参数总数

**33 个参数** = Zero-Shot CoT (27个) + 检索器参数 (3个) + 分块参数 (3个)

## 方法四：ReAct

ReAct 方法**继承 RAG CoT 的所有参数**，并增加以下特定参数：

### ReAct 特定参数

通过 `run.strategy_params` 或 `configs/methods/react.yaml` 配置：

| 参数路径 | 类型 | 默认值 | 说明 |
|----------|------|--------|------|
| `run.strategy_params.max_steps` | int | 6 | ReAct 最大推理步数（Thought-Action-Observation 循环次数） |

### 必需配置差异

| 配置项 | 值 | 说明 |
|--------|-----|------|
| `run.strategy` | "react" | 必须设置为 react |
| `run.strategy_params.max_steps` | 6 | ReAct 最大步数 |
| `model.batch_size` | 1 | 自动强制为 1（交互式策略） |

### 运行特点

- **交互式推理**：执行 Thought（思考）→ Action（行动）→ Observation（观察）循环
- **支持的动作**：
  - `Search[entity]`：检索相关文档
  - `Lookup[keyword]`：在当前文档中查找关键词
  - `Finish[answer]`：给出最终答案
- **批处理限制**：由于是交互式策略，`batch_size` 自动设置为 1
- **Few-shot 提示**：使用预定义示例引导模型执行 ReAct 推理

### 参数总数

**34 个参数** = RAG CoT (33个) + ReAct 特定参数 (1个)

## 参数使用建议

### 性能优化

1. **内存不足时**：
   - 减少 `batch_size`
   - 减少 `max_new_tokens`
   - 减少 `retriever.top_k`
   - 减少 `chunk.chunk_size`

2. **提高速度**：
   - 增加 `batch_size`（非 ReAct 方法）
   - 使用 `model.provider="vllm"`
   - 减少 `retriever.top_k`
   - 减少 `react.max_steps`

3. **提高准确率**：
   - 增加 `retriever.top_k`
   - 增加 `react.max_steps`
   - 调整 `model.temperature` (0.0 = 贪婪，>0 = 采样)
   - 使用更强的模型（llama_70b, gpt-4）

### 调试建议

1. **快速测试**：
   - 设置 `run.max_questions=10`
   - 使用较小的模型
   - 设置 `retriever.top_k=3`

2. **完整评估**：
   - 设置 `run.max_questions=0`（处理全部）
   - 启用 `run.resume=true`（支持中断恢复）
   - 设置 `run.save_every=1`（及时保存）

3. **结果复现**：
   - 固定 `run.seed`
   - 固定 `model.temperature=0.0`
   - 记录所有参数配置

## 配置文件示例

### MRAG 运行示例

```bash
cd /workspace/ETE-Graph/agent/MRAG-master
python reader.py \
    --max-examples 100 \
    --retriever-output "situatedqa_contriever_metriever_bgegemma_llama_8b_qfs5_outputs.json" \
    --ctx-topk 3 \
    --stage1-model contriever \
    --reader gpt \
    --paradigm concat \
    --param-pred True
```

### Zero-Shot CoT 配置示例 (YAML)

```yaml
data:
  corpus_path: "/path/to/corpus.json"
  question_arrow_path: "/path/to/questions.arrow"

model:
  provider: "local"
  model_name: "Qwen/Qwen3-4B-Instruct-2507"
  max_new_tokens: 256
  temperature: 0.0
  batch_size: 4

run:
  strategy: "zero_shot_cot"
  max_questions: 100
  resume: true
```

### RAG CoT 配置示例 (YAML)

```yaml
data:
  corpus_path: "/path/to/corpus.json"
  question_arrow_path: "/path/to/questions.arrow"

model:
  provider: "vllm"
  model_name: "qwen3-32b"
  base_url: "http://localhost:8000/v1"

retriever:
  type: "contriever"
  model_name: "facebook/contriever"
  top_k: 5

chunk:
  chunk_size: 1200
  chunk_overlap: 100

run:
  strategy: "rag_cot"
  max_questions: 100
```

### ReAct 配置示例 (YAML)

```yaml
data:
  corpus_path: "/path/to/corpus.json"
  question_arrow_path: "/path/to/questions.arrow"

model:
  provider: "remote"
  model: "gpt-4"
  base_url: "https://api.openai.com/v1"
  api_key_env: "OPENAI_API_KEY"

retriever:
  type: "contriever"
  top_k: 5

run:
  strategy: "react"
  max_questions: 50
  strategy_params:
    max_steps: 6
```
